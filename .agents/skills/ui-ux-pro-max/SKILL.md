# ui-ux-pro-max

Source: `https://github.com/nextlevelbuilder/ui-ux-pro-max-skill`

## When to use

Use this skill when the task changes how UI looks, feels, or behaves:

- New pages/screens and navigation structures.
- Component creation/refactoring.
- Design-system choices (colors, type, spacing, states).
- UI/UX quality reviews and consistency checks.

## Working mode

1. Start with product context and user goals.
2. Define/confirm design tokens and component rules.
3. Implement UI using the repository design system conventions.
4. Run accessibility and interaction checks before handoff.

## Guard rails

- Prefer existing project design-system APIs over ad-hoc styling.
- Avoid decorative complexity without product value.
- Enforce readable contrast, clear focus states, and touch-safe targets.
- Keep responsive behavior predictable across common breakpoints.
