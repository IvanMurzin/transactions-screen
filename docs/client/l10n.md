# Localization

Two languages: English (`app_en.arb`) and Russian (`app_ru.arb`).
Both files must contain the same set of keys at all times.

## Adding a string

1. Add the key to **both** ARB files in the same commit.
2. Use it: `AppLocalizations.of(context)!.<key>`.
3. Run `flutter pub get` to regenerate `app_localizations*.dart` (or
   it'll auto-regen on the next `flutter run`).

## Plurals + placeholders

```json
{
  "@@locale": "en",
  "itemsCount": "{count, plural, =0{no items} =1{1 item} other{{count} items}}",
  "@itemsCount": {
    "placeholders": {
      "count": { "type": "int" }
    }
  }
}
```

## Switching language at runtime

`LocaleCubit` (`core/localization/locale_cubit.dart`) owns the
choice; `LocaleStorage` persists it. `null` means "follow device
locale".

## Server-side translations

If your backend returns user-facing error messages, translate them on
the client side using a small lookup table. Don't ship localized
strings from the backend.
