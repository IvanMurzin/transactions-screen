# Product docs

Source of truth for *what* the app exists to do and *for whom*.
Architecture and code answer "how"; these docs answer "why" and decide
trade-offs when the code path is ambiguous.

The `create-product` skill walks you through filling each file by
interviewing you. Run it once when you start a new app. Update the
files as the product evolves; never let them go stale relative to what
ships.

| File | Purpose |
| ---- | ------- |
| [`product_brief.md`](product_brief.md) | One-paragraph elevator pitch, vision, and the "why now". |
| [`audience_and_jtbd.md`](audience_and_jtbd.md) | Target users, their jobs-to-be-done, current alternatives. |
| [`problem_solution.md`](problem_solution.md) | Specific pain points and how this product addresses them. |
| [`mvp_scope.md`](mvp_scope.md) | What's in v1, what's deferred. Hard line. |
| [`business_model.md`](business_model.md) | How the product makes money (or doesn't). |
| [`financial_model.md`](financial_model.md) | Unit economics, projected MRR, breakeven assumptions. |
| [`acquisition_and_growth.md`](acquisition_and_growth.md) | Channels, costs, conversion assumptions. |
| [`retention.md`](retention.md) | Why users come back; what would make them leave. |
| [`metrics.md`](metrics.md) | The 3-7 numbers that decide whether the product is working. |
| [`risks.md`](risks.md) | Top risks (market, tech, regulatory) and mitigations. |
| [`roadmap.md`](roadmap.md) | Sequenced bets beyond MVP. |
| [`open_questions.md`](open_questions.md) | Decisions still pending and what blocks them. |
