# Design system overview

## Philosophy

The design system is **created per project**, not inherited from the template.
Generic placeholder components lead to generic-looking apps. This template
ships only the minimal infrastructure (theme switching, Material 3 baseline)
— the actual tokens, palette, typography, and components are generated fresh
for each project by `ui-ux-pro-max`.

## How to create the design system for a new project

Run `/setup-design-system` after `/create-product`.

The skill reads your product docs, invokes `ui-ux-pro-max` with full
product context, and generates:

| Output | Location |
|--------|----------|
| Token classes (DSColors, DSTypography, DSSpacing, DSRadius, DSElevation) | `client/lib/core_ui/theme/ds_theme.dart` |
| Light + dark ThemeData wiring | `client/lib/core_ui/theme/app_theme.dart` |
| 14 core components | `client/lib/core_ui/components/` |
| DS preview route | `client/lib/core_ui/preview/ds_preview_page.dart` |
| Design doc | `docs/design_system/system.md` |

See [setup-design-system skill](../../.claude/skills/setup-design-system/SKILL.md).

## What ships in the template (baseline only)

| File | Purpose |
|------|---------|
| `theme/app_theme.dart` | Bare Material 3 `lightTheme` / `darkTheme` from a seed color |
| `theme/theme_mode_cubit.dart` | Persisted light/dark/system toggle |

Everything else (`ds_theme.dart`, all components, preview page) is
generated by `/setup-design-system` and is **not** in the template.

## DS contract (after setup-design-system runs)

All feature code must:
- Use `context.dsColors`, `context.dsSpacing`, `context.dsTypography` etc.
  (BuildContext extensions defined in `ds_theme.dart`)
- Use components from `core_ui/components/` — never raw `Color(0x…)` or
  hard-coded padding values in feature widgets
- Use the `/design-system` route to verify component appearance before shipping

## DS audit

Run `/review-core-ui` to check token coverage, light/dark parity,
component inventory, and feature-code DS adoption.

## Screen design docs

Per-area design docs (screen wireframes, component specs) are saved to
`docs/design_system/screens/<area>.md` by `/create-all-specs` and
`/create-spec`. Specs reference these via the `design_doc` frontmatter field.
